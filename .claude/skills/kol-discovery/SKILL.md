# KOL Discovery

Load at the start of every KOL discovery session. Provides the full KOL methodology — discovery, quality evaluation, required fields, and Notion write rules.

---

## Core Principles

- Always start from a high-quality seed or user-provided handle — not a search bar
- Quality compounds: high-quality KOLs cluster around other high-quality KOLs
- Hashtag search is outdated in 2026 — not a primary discovery method
- Instagram natural language keyword search is the correct search method
- One tab, navigate within it — never open a new tab for each profile

---

## Exclusion List — Build Before Any Discovery Begins

Read `context/kol-exclusion-list.md`. Load every handle into an in-memory exclusion list for this session. Before researching or writing any candidate, check this list — if the handle is here, skip immediately. Each time a new record is written to Notion, add that handle to the in-memory list immediately.

The file is generated by `node tools/sync-kol-list.js` (Notion REST API with full pagination — no cap). Run it at the start of every KOL session to ensure the list is current. Do not query Notion directly for dedup — the search tool caps at 25 results and will miss records.

## Discovery Methods

**Step 0 — User-provided seeds (always first):**
The user sometimes shares handles they've saved or want to evaluate. Always check these first. Even if some are off-market, they define the aesthetic and style benchmark. Evaluate each against C1–C6 before proceeding to independent discovery.

**Method 1 — Social graph expansion:**
From each confirmed seed or qualified KOL, run four expansion passes:
- **Pass A (Mutuals):** Scan 50–100 accounts from their Following list — same-niche creators follow each other.
- **Pass B (Comment section):** Open their last 10–15 posts. Look for creators leaving substantive comments (not emoji), accounts the KOL replies to (reciprocal engagement = real relationship), recurring commenters across multiple posts.
- **Pass C (Tagged content):** Check tagged posts. Creators who collab-post or tag have already demonstrated collaboration willingness.
- **Pass D (Suggested accounts):** Instagram's "Similar Accounts" on the profile — pre-filtered by Instagram's own interest clustering. Review first 10–20 suggestions.

**Method 2 — Competitor KOL reverse-engineering:**
For every competitor in scope, find who is already creating content about them. Search each competitor brand name on Instagram, YouTube, X, and Reddit. Find who tags competitors in posts, mentions them in comments, or features them in reviews. If they fit the buyer persona but don't overlap with the brand's positioning territory, they're candidates.

**Method 3 — Keyword-based profile search (Instagram):**
Use natural language keyword phrases in Instagram search — not hashtags. Use: content niche terms, lifestyle terms matching the buyer persona, location + niche combinations for market-specific discovery. Ignore brand accounts and media accounts. Focus on individual creators in nano/micro range.

**Method 4 — Community hubs:**
Reddit top contributors in relevant subreddits. YouTube search results for category keywords — who appears consistently. Look for users who reference their own content, link to their channels, or demonstrate genuine category expertise.

## Platform-Specific Discovery Intelligence

### Instagram — Primary Discovery Platform

**Discovery order:**
1. User-provided seeds first — evaluate these before any independent discovery
2. Doomscroll from seeds — navigate to a seed's profile via Chrome DevTools MCP, then explore: their following list, suggested similar accounts, accounts tagged in their posts, "suggested for you." Follow the trail organically, like a human talent scout
3. Doomscroll from already-qualified KOLs in the current run
4. Web search as supplement — search creator name + "Instagram" or niche + `site:instagram.com` to find adjacent creators. Supplement only — not the primary method
5. Hashtags — outdated, not reliable. Skip

**Tool:** Chrome DevTools MCP (`mcp__chrome__*`) — user's logged-in Chrome. Never use `isolatedContext`. Never open a new tab for each profile — one tab, navigate within it each time.

**Hashtag abandonment rule:**
Niche product category hashtags are frequently dominated by retail store accounts. If 2 consecutive posts from a hashtag belong to store or brand accounts → abandon the hashtag immediately. Do not continue.

**Private account early detection:**
Read only the first 60–70 lines of any profile snapshot. If "This profile is private" appears → skip immediately. No further assessment needed.

**Snapshot reading efficiency:**
Never read a full profile snapshot (200+ lines). The first 50–60 lines contain everything needed: follower count, bio, email, UpDog ER overlays on the first few posts. Everything below line 100 is typically just the language selector dropdown — skip it.

### Instagram — Low-Yield Markets

Some markets have low Instagram creator yield for niche product categories. This is common where YouTube or TikTok is the dominant creator platform (e.g., Southeast Asian markets for tech and lifestyle). Signals of a low-yield Instagram market:
- Repeated store accounts appearing in hashtag and search results
- Few personal creator accounts with genuine audiences
- Creators present but with low follower counts and minimal engagement

**Adjusted approach for low-yield markets:**
- Web search first — identify named candidates before opening Instagram. Query: `"[country] Instagram influencer [product category]"`. Pre-qualify candidates by name before visiting profiles.
- Use Instagram only for verification — open a profile only after a candidate is named from external sources.
- Never browse blind — if 2 consecutive profiles are store accounts, stop and use web search to re-seed.
- Plan for lower yield per hour and adjust time budget accordingly.

## KOL Quality Standards

**Tiers:** Nano (<10K) · Micro (10K–100K) · Mid-Tier (100K–500K) · Macro (500K–1M) · Mega (1M+)

### ER Formulas

**YouTube ER (view-based — the only correct formula):**
- Formula: `(Avg Likes + Avg Comments) ÷ Avg Views × 100`
- Window: last 10 videos published within the last 12 months. "1 year ago" label is borderline-included; "2 years ago" is excluded. If fewer than 10 qualify, use all available within 12 months.
- Collect views, likes, and comments from each individual video page. Average across all qualifying videos, then apply the formula.
- DO NOT use views ÷ subscribers — that is a view rate, not ER. It produces numbers above 100% that are meaningless.

**Instagram ER (UpDog):**
- UpDog calculates ER from the last 30 days of posts.
- Read via JS DOM query: `document.querySelector('.updog').innerText` — not the a11y tree (values don't appear there).
- One UpDog read returns all four values at once: ER, Avg Views, Avg Likes, Avg Comments. Collect and record all four on every profile visit — never capture only ER and leave the rest blank. Leave Average Views blank only if UpDog returns "--" (statics-only account).
- UpDog works on Instagram only. Do not attempt on YouTube or other platforms.

**ER pass/fail gate:**
- Instagram ≥1% — fail = skip immediately, do not assess further
- YouTube ≥1% — fail = skip immediately, do not assess further

**Benchmarks:** <1% Low · 1–3.5% Average · >3.5% High

**ER overlay anomalies:**
- Two anomalously large numbers on the first two posts (e.g., 4223%, 280%) = pinned viral outliers. Ignore these. Read posts 3–10 for representative ER.
- If recent posts consistently show <1%, the creator fails the ER gate regardless of third-party estimates.

### Country Verification

**Instagram — definitive method:**
Navigate to the profile → tap the three-dot menu → "About this account" → read the "Account based in" field. This is the registered country and the primary definitive signal. It takes precedence over bio text, flag emojis, caption language, and all other signals. If "Account based in" shows a different country than the target market → the creator fails C2 immediately. Skip.

After confirming country via "About this account", use caption language, comment language and slang, and story location tags as supporting signals only.

**YouTube:**
Channel "About" page country field, or API `country` field if using the script. If "Not set" — verify via bio signals and caption language in browser.

### Quality Criteria — C1 through C6

**C1 — Engagement rate (pass/fail gate):**
Apply ER formulas above. Fail = skip immediately. Do not assess further.

**C2 — Audience demographics vs. target persona:**
Does this creator's audience match the target buyer persona — country (verified above), language, lifestyle signals?

**C3 — Purchase-intent signals in comments:**
Scroll 20–30 comments across 3–5 recent posts. Look for: "just ordered," "where can I buy," "how much is this," detailed spec questions. High purchase-intent comment frequency = strong signal.

**C4 — Sponsored content performance:**
Has the creator done branded content? Does it perform at a similar level to their organic content? If sponsored posts have <50% of normal engagement — disqualifying.

**C5 — Growth trajectory:**
Is the creator growing, flat, or declining? Sudden spikes followed by flat periods indicate bought engagement — flag and verify.

**C6 — Verbatim language alignment:**
Does the creator's own language — captions, commentary, self-description — align with the verbatim language map from buyer research? A creator who naturally uses the same words your buyer uses is a higher fit than one who doesn't, regardless of follower count.

### Content Quality Filter

- **Visual quality:** Lighting, color grading, composition — intentional? Natural lighting handled well counts equally to studio.
- **Consistency:** Does quality hold across the last 9–12 posts?
- **Authenticity:** Does the content feel like a genuine person's perspective?

A nano creator with consistently high-quality content outweighs a micro creator with one viral post and mediocre surrounding content.

### YouTube API Script

**Before any YouTube KOL data collection:** Check whether `.env` has a `YOUTUBE_API_KEY` entry.
- If it exists → use the API script (faster, more reliable): `node tools/youtube-kol-data.js @Handle1 @Handle2 @Handle3`
- If it does not → collect data manually via browser

**What the script returns per KOL:** `subscribers`, `country`, `er`, `erPass` (true/false ≥1%), `avgViews`, `avgLikes`, `avgComments`, `videos` (per-video breakdown), `c3Comments` (50 comments from top video for C3 assessment).

**What the script does NOT replace:** email lookup, content quality check (C2, C4, C5, C6), country verification if "Not set".

**Handle verification — exact match required:**
Before concluding a channel doesn't exist, verify the exact handle character-by-character. Try at minimum: exact provided handle, trailing underscore variant, different capitalisation. A channel returning empty or zero subscribers likely has a handle mismatch — do not mark it as non-existent without trying variants.

### Required Fields Before Any Notion Write

Never write a KOL record until every field below is populated:

- **Followers/Subscribers**
- **ER** (verified — not estimated from third-party tools)
- **Average Views**
- **Average Likes** (Instagram: from UpDog; YouTube: from individual video pages or API)
- **Average Comments** (Instagram: from UpDog; YouTube: from individual video pages or API)
- **Contact/Email** — attempt lookup: platform About page → web search `"[creator name] email collab contact"` → bio links/Linktree. If genuinely unavailable after all attempts, write "Not found"
- **Country** (Instagram: verified via "About this account" → "Account based in". YouTube: from channel "About" page or API `country` field)
- **Links** (the KOL's own platform URL — not a third-party article)
- **Agent Reason** — 1–2 sentences. Format: how they were found + why they are relevant for the brief. Never leave blank. Never write more than 2 sentences.

**Tags and Description — never populate.** These are the user's manual curatorial decisions. Leave both blank.

### Notion Write Rules

- **No duplicates:** Read `context/kol-exclusion-list.md` at the start of every discovery task. Hard-exclude all listed handles before any discovery begins. Never query Notion directly for this.
- **Immediate writes:** Write each confirmed KOL to Notion immediately after they pass all criteria. Never batch at the end. Never accumulate.
- **Write verification required after each write:** Fetch the record back by its Notion URL to confirm the write succeeded. If the fetch fails → stop immediately and report.
- **Update exclusion list:** Add the handle to the in-memory exclusion list immediately after writing, so duplicates within the same run are blocked.
